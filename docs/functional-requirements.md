# 機能要件書

## プロジェクト概要

**プロジェクト名**: Menu Scanner for International Visitors  
**目的**: AI を使用して、アレルギーや宗教的制約を持つ外国人観光客が日本の飲食店で安心して食事を楽しめるアプリケーション  
**対象デバイス**: スマートフォン（Web アプリケーション）  
**技術スタック**: React + Vite, Firebase, Cloud Vision API, Gemini LLM

## 機能要件

### 1. ユーザー管理機能

#### 1.1 ユーザー認証

- **機能 ID**: F001
- **概要**: Firebase Authentication を使用したユーザー登録・ログイン
- **詳細**:
  - メールアドレス・パスワードによる認証
  - Google アカウント連携（オプション）
  - ゲストモード対応（一時的な利用）
- **優先度**: 必須

#### 1.2 プロフィール管理

- **機能 ID**: F002
- **概要**: ユーザーの食事制約情報の登録・管理
- **詳細**:
  - アレルギー情報の登録（複数選択可能）
    - ナッツ類、甲殻類、卵、乳製品、小麦、大豆、魚類など
  - 宗教的制約の登録
    - 豚肉、牛肉、アルコール、ハラル対応など
  - カスタム制約の追加機能
  - プロフィール情報の編集・削除
- **優先度**: 必須

### 2. メニュー解析機能

#### 2.1 画像取り込み

- **機能 ID**: F003
- **概要**: カメラまたはギャラリーからメニュー画像を取得
- **詳細**:
  - リアルタイムカメラ撮影
  - ギャラリーからの画像選択
  - 画像のトリミング・回転機能
  - 複数画像の一括処理
- **優先度**: 必須

#### 2.2 OCR 処理

- **機能 ID**: F004
- **概要**: Cloud Vision API による日本語テキスト抽出
- **詳細**:
  - 日本語（ひらがな、カタカナ、漢字）の認識
  - 英語併記メニューの対応
  - テキスト抽出結果の表示・編集機能
  - 認識精度の向上のための前処理
- **優先度**: 必須

#### 2.3 食材推論

- **機能 ID**: F005
- **概要**: Gemini LLM による料理名からの食材推論
- **詳細**:
  - 日本料理の食材構成の推論
  - 調理法・調味料の推定
  - 料理の一般的なバリエーション情報
  - 推論結果の信頼度表示
- **優先度**: 必須

### 3. 食材チェック機能

#### 3.1 制約チェック

- **機能 ID**: F006
- **概要**: ユーザーの制約情報とメニューの食材を照合
- **詳細**:
  - アレルギー食材の検出
  - 宗教的制約の確認
  - リスクレベルの分類（安全/注意/危険）
  - 代替提案の表示
- **優先度**: 必須

#### 3.2 結果表示

- **機能 ID**: F007
- **概要**: チェック結果の視覚的な表示
- **詳細**:
  - 色分けによる安全度表示（緑/黄/赤）
  - 料理ごとの詳細情報表示
  - 注意すべき食材のハイライト
  - 推奨度の表示
- **優先度**: 必須

#### 3.3 店員向け説明文生成

- **機能 ID**: F008
- **概要**: ユーザーの制約を店員に伝える文章の自動生成
- **詳細**:
  - 日本語・英語での説明文生成
  - 視覚的にわかりやすい表示
  - 印刷・共有機能
  - カスタマイズ可能なテンプレート
- **優先度**: 必須

### 4. 文化情報提供機能

#### 4.1 食べ方ガイド

- **機能 ID**: F009
- **概要**: 日本料理の正しい食べ方の説明
- **詳細**:
  - 寿司、そば、ラーメンなどの食べ方
  - AI 画像生成による視覚的説明
  - ステップバイステップのガイド
  - 動画・アニメーション対応
- **優先度**: 推奨

#### 4.2 文化背景説明

- **機能 ID**: F010
- **概要**: 料理の歴史・文化的背景の紹介
- **詳細**:
  - 料理の起源・歴史
  - 地域性・季節性の説明
  - 関連する日本文化の紹介
  - 多言語対応
- **優先度**: 推奨

### 5. 履歴・お気に入り機能

#### 5.1 スキャン履歴

- **機能 ID**: F011
- **概要**: 過去にスキャンしたメニューの履歴管理
- **詳細**:
  - スキャン日時・場所の記録
  - 結果の再表示
  - 履歴の検索・フィルタリング
  - 履歴の削除機能
- **優先度**: 推奨

#### 5.2 お気に入り登録

- **機能 ID**: F012
- **概要**: 安全な料理・レストランのお気に入り登録
- **詳細**:
  - 料理・レストランのブックマーク
  - メモ・評価の追加
  - お気に入りの共有機能
  - カテゴリ分け
- **優先度**: オプション

### 6. 設定・その他機能

#### 6.1 アプリ設定

- **機能 ID**: F013
- **概要**: アプリケーションの各種設定
- **詳細**:
  - 言語設定（日本語/英語）
  - 通知設定
  - プライバシー設定
  - データ同期設定
- **優先度**: 必須

#### 6.2 ヘルプ・サポート

- **機能 ID**: F014
- **概要**: ユーザーサポート機能
- **詳細**:
  - 使い方ガイド
  - FAQ
  - お問い合わせフォーム
  - チュートリアル
- **優先度**: 推奨

## 非機能要件

### パフォーマンス要件

- **レスポンス時間**: OCR 処理は 10 秒以内、LLM 処理は 15 秒以内
- **同時接続数**: 100 ユーザー同時利用可能
- **可用性**: 99.5%以上

### セキュリティ要件

- **データ暗号化**: 通信・保存データの暗号化
- **個人情報保護**: GDPR・個人情報保護法準拠
- **認証**: Firebase Authentication による安全な認証

### 互換性要件

- **ブラウザ対応**: Chrome, Safari, Firefox の最新版
- **OS 対応**: iOS 12+, Android 8+
- **画面サイズ**: 320px〜768px（スマートフォン）

## 制約事項

### 技術的制約

- Firebase での運用
- Cloud Vision API の月間利用制限
- Gemini API のレート制限

### 運用制約

- 日本語メニューのみ対応（第一段階）
- オフライン機能は限定的
- リアルタイム翻訳は対象外

## 成功指標

### 定量指標

- **ユーザー登録数**: 月間 100 ユーザー
- **スキャン成功率**: 85%以上
- **ユーザー満足度**: 4.0/5.0 以上

### 定性指標

- 外国人観光客の日本食体験の向上
- 飲食店でのコミュニケーション円滑化
- 食の安全性確保への貢献

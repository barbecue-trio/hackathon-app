# UI 設計書

## 設計方針

### 対象デバイス

- **プライマリ**: スマートフォン（Web アプリケーション）
- **画面サイズ**: 320px〜768px
- **向き**: 縦向き（ポートレート）メイン

### デザインコンセプト

- **ユーザビリティファースト**: 外国人観光客でも直感的に操作できる
- **視覚的コミュニケーション**: 言語の壁を越える絵文字・アイコンの活用
- **安心感の提供**: 食の安全性を色とアイコンで明確に表現
- **日本らしさ**: 温かみのある色彩で日本の「おもてなし」を表現

## カラーパレット

### メインカラー

- **プライマリ**: `#E53E3E` (温かみのある赤 - 日本らしさ)
- **セカンダリ**: `#D69E2E` (金色 - 高級感・特別感)
- **アクセント**: `#3182CE` (青 - 信頼感)

### ステータスカラー

- **安全**: `#38A169` (緑 - 食べられる)
- **注意**: `#D69E2E` (オレンジ - 要確認)
- **危険**: `#E53E3E` (赤 - 避けるべき)
- **情報**: `#3182CE` (青 - 一般情報)

### ベースカラー

- **背景**: `#FFFFFF` (白)
- **サブ背景**: `#F7FAFC` (薄いグレー)
- **テキスト**: `#2D3748` (ダークグレー)
- **サブテキスト**: `#718096` (ミディアムグレー)

## タイポグラフィ

### フォント

- **メイン**: システムフォント (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto`)
- **日本語**: `'Hiragino Sans', 'Yu Gothic', sans-serif`
- **英語**: `'Inter', 'Helvetica Neue', sans-serif`

### フォントサイズ

- **見出し 1**: 24px (1.5rem)
- **見出し 2**: 20px (1.25rem)
- **見出し 3**: 18px (1.125rem)
- **本文**: 16px (1rem)
- **キャプション**: 14px (0.875rem)
- **小文字**: 12px (0.75rem)

## レスポンシブ対応

### ブレークポイント

- **Small**: 320px - 480px (小型スマートフォン)
- **Medium**: 481px - 768px (大型スマートフォン・小型タブレット)

### レイアウト調整

```css
/* Small devices */
@media (max-width: 480px) {
  .container {
    padding: 12px;
  }
  .grid-2x2 {
    grid-template-columns: 1fr;
  }
  .font-size {
    font-size: 14px;
  }
}

/* Medium devices */
@media (min-width: 481px) {
  .container {
    padding: 16px;
  }
  .grid-2x2 {
    grid-template-columns: 1fr 1fr;
  }
  .font-size {
    font-size: 16px;
  }
}
```

## アクセシビリティ

### 基本要件

- **コントラスト比**: WCAG AA 準拠（4.5:1 以上）
- **フォーカス表示**: キーボードナビゲーション対応
- **タッチターゲット**: 最小 44px×44px
- **スクリーンリーダー**: aria-label, alt 属性の適切な設定

### 多言語対応

- **言語切り替え**: ヘッダーに言語選択ボタン
- **RTL 対応**: アラビア語等の右から左への言語対応
- **フォント**: 各言語に適したフォントの選択

## アニメーション・インタラクション

### マイクロインタラクション

- **ボタンホバー**: 0.2 秒のスムーズな色変化
- **カード表示**: フェードイン効果
- **ローディング**: スピナーアニメーション
- **成功/エラー**: トーストメッセージ

### ページ遷移

- **スライド**: 左右のスライドアニメーション
- **フェード**: 透明度の変化
- **所要時間**: 0.3 秒以内

## パフォーマンス考慮事項

### 画像最適化

- **形式**: WebP 優先、フォールバックで JPEG/PNG
- **サイズ**: レスポンシブ画像の実装
- **遅延読み込み**: Intersection Observer API 使用

### CSS 最適化

- **Critical CSS**: Above-the-fold CSS の優先読み込み
- **CSS Grid/Flexbox**: レイアウトの効率化
- **CSS Variables**: テーマ切り替えの効率化

この設計により、外国人観光客にとって使いやすく、視覚的にわかりやすいスマートフォン Web アプリケーションを実現できます。
